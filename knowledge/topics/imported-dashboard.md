# Dashboard
## Overview
The Dashboard is the default landing page when logging into the CMS platform. It provides a high-level summary of your entire digital signage ecosystem — allowing quick monitoring of device health, content scheduling status, and storage usage, all in one centralized view.
## Key Dashboard Features
### Device Status Overview(Green, Red, Teal Boxes)
Located at the top-left of the dashboard, this section gives a quick summary of device connectivity:
- Online Devices (Green Box)
Devices currently connected and functioning normally.
Clicking this opens a filtered view of online devices.
- Offline Devices (Red Box)
Devices that are unreachable or not reporting status.
Action is required to investigate disconnection.
- Unpaired Devices (Teal Box)
Devices that have not yet been linked to a network.
Typically appear after fresh installs or factory resets.
### Device Table List
Located under the device status boxes, this searchable table allows you to:
- View all devices by name and current status
- Filter by network or keyword
- Identify individual device issues (e.g., status icons: green = online, red = offline, orange = unpaired/warning)
- Use pagination controls to navigate large inventories
### Online Network Visualization(Blue Circle)
This visual graph on the right shows how many devices are online by network group. Hover or click to view the breakdown. Useful for:
- Spotting active vs. inactive networks
- Tracking regional or client-based uptime
### Storage Usage Panel(Top Right)
This bar shows how much CMS storage is currently used out of your total limit (e.g., 4.22GB / 5GB).
It helps monitor:
- Media storage capacity
- Upload planning
- Cleanup needs to prevent overages
### Notifications Panel(Red badge icon in top nav)
Access alerts related to:
- Pending Content: Awaiting approval or review
- Placeholder Content: Fallback content needing replacement
- Instant Events: Quick Play-triggered real-time messages
### Release Information Widget
Stay updated with the latest software and player updates:
- Release Notes: Recent CMS or player improvements
- Player Build Version: Version currently installed on your screens
- CMS Platform Version: Current backend version in use
- More Button: Link to full release changelog
## Best Practices for Using the Dashboard
- Review the dashboard daily to detect issues early
- Investigate any offline or unpaired devices via Device Manager
- Monitor storage usage to plan future uploads
- Resolve pending or placeholder content to avoid screen blackouts
- Track software version updates to ensure compatibility and performance
