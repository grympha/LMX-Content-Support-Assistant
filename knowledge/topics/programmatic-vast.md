# Programmatic / VAST

Programmatic and VAST content require correct ad configuration, fallback content, compatible platform, modern browser/WebView, and stable internet.

## Steps

1. Confirm device platform supports URL, VAST, HTML, or programmatic playback.
2. Check WebView or browser requirements.
3. Create the URL, Google IMA/VAST, Hivestack, or programmatic schedule.
4. Set ad slot duration.
5. Add filler or fallback content.
6. Enter required API key, organization ID, screen ID, or ad unit ID.
7. Save the schedule.
8. Publish to relevant playlist or devices.
9. Validate playback.

## Important Notes

- Windows and Android 11+ are preferred for programmatic-heavy networks.
- Android WebView should be modern.
- Filler content is important when ad delivery has no fill or fails.
- Network must allow access to ad endpoints.

## Common Mistakes

- No filler content.
- Wrong screen ID or ad unit ID.
- Unsupported OS or WebView.
- Network blocks ad endpoints.
