# MongoDB M30 to M40 Upgrade

## Incident Summary

Platform performance degradation observed during peak operations.

## Impact

- Slow CMS response
    
- Delayed content deployment
    
- Increased loading times
    

## Investigation

Infrastructure monitoring identified database resource constraints.

## Root Cause

MongoDB cluster resources were insufficient for increasing platform load.

Current Cluster:

- MongoDB M30
    

## Resolution

Infrastructure upgraded:

- MongoDB M30 → MongoDB M40
    

## Result

- Improved performance
    
- Reduced latency
    
- Better deployment stability
    

## Lessons Learned

- Monitor database growth proactively.
    
- Review infrastructure capacity regularly.
